    How the three-tier filter works

    1. Ignored extensions - Completely allowed, no action taken
    2. Warn extensions - A warning is sent, but the message is not deleted
    3. All other extensions - Message is deleted and a warning is sent
    Note: Users with "Manage Messages" permission are exempt from deletion - they only receive warnings.

    Filter Status

    Enable or disable the attachment filter
    When enabled, the bot will check all message attachments against the configured tiers.

    Ignored Extensions (Allowed)

    These file types are completely allowed - no action taken
    Common safe extensions: .png .jpg .jpeg .gif .webp .pdf .txt .mp4 .mp3

    Warn Extensions (Caution)

    These file types trigger a warning but are not deleted
    Use for potentially risky but sometimes needed files: .zip .rar .7z .doc .docx .xls .xlsx

    Blocked Extensions (Everything Else)

    All extensions not listed above will be blocked
    Any file extension not in the Ignored or Warn lists will be deleted and a warning sent.

    Examples of commonly blocked dangerous extensions:
    .exe .bat .cmd .scr .msi .vbs .js .jar .ps1 .dll

    Warning Message

    Customize the message sent for warned and blocked files
    Available placeholders: {user} (mentions user), {extension} (file extension), {filename} (filename), {action} ("removed" or "flagged")
    Default blocked message:
    @user, your message was removed because the file type (.exe) is not allowed. Please use an approved file format.
    Default warn message:
    @user, your attachment (.zip) requires caution. Please ensure you trust the source of this file.
    Example custom message:
    {user} - The file "{filename}" ({extension}) was {action}. Contact a moderator if you need to share this file type.
